# 恋愛モビー診断 共有カード仕様書

## 1. 共通仕様

### 用途

- 診断結果のSNS共有カード
- BURN内での結果表示
- プロフィール帳風の結果ページ
- 将来的な印刷・雑誌化のベース

### サイズ

- A4縦
- 推奨比率：1414:2000 前後
- SNS投稿用にトリミングしやすい縦長設計

### デザイントーン

- ロマンチック
- 上品
- かわいい
- ふんわり
- 少し雑誌っぽい
- 診断結果だけど保存したくなる見た目

### 共通レイアウト構成

上から順に以下の構成にする。

1. 診断名
2. 結果導入文
3. タイプ名
4. タイプコードバッジ
5. サブコピー
6. キャラ画像メイン表示
7. 「あなたはこんな人！」
8. 「恋のキーワード」
9. 「あなたの恋愛スタイル」
10. 「こんな人と相性がいいよ」
11. 「あなたの魅力」
12. 「もっと素敵な恋をするためのヒント」
13. 最下部メッセージ

### 下部固定文言

```text
あなたの記憶がいつまでも鮮やかに豊かであり続けますように
```

### 削除対象

共有カードには以下は入れない。

- QRコード
- SNSアイコン
- シェア誘導文言
- ABCD軸表記
- 4 AXES表記

## 2. キャラクター画像使用ルール

### 基本ルール

共有カード内のキャラクターは、画像生成で新規作成しない。  
各タイプ名に対応する完成済みキャラクター画像ファイルを使用する。

### 画像ファイルの参照場所

キャラクター画像は以下のディレクトリ内にあるものを参照する。

```text
docs/image/lovemobby/
```

### 使用方法

- タイプコードではなく、タイプ名に対応する画像ファイルを参照する。
- 共有カード生成時は、診断結果の `typeName` に対応する `characterImageFile` を参照する。
- キャラクター画像は、カード中央のメインビジュアルとして配置する。
- キャラクターの造形・表情・小物・色は、対応画像ファイル側を正とする。
- カード側では、キャラクター画像を改変しない。
- 必要な調整は、拡大縮小・位置調整・影・背景とのなじませ処理までにする。
- 画像の差し替えが必要な場合は、同じタイプ名の画像ファイルを `docs/image/lovemobby/` 内で更新する。

### 画像ファイル管理

各タイプの個別仕様には、以下の項目を持たせる。

```text

```

## 3. 診断軸

| 軸 | 左方向 | 右方向 | 意味 |
|---|---|---|---|
| A軸 | H = Heroine / ヒロイン型 | S = Support / 支え役型 | 恋愛の中で、自分が主役寄りか、支える側寄りか |
| B軸 | L = Lock / そばにいたい型 | F = Free / 自由もほしい型 | 恋人と強く結びつきたいか、自由も大事にしたいか |
| C軸 | T = Tokimeki / ときめき重視型 | A = Anshin / 安心重視型 | 恋にときめきを求めるか、安心を求めるか |
| D軸 | O = Open / 見せたい恋型 | C = Close / 内にしまう恋型 | 恋愛を外に見せたいか、二人の内側で大事にしたいか |

## 6. 実装・量産用補足

### カードに必ず入れる固定見出し

- あなたはこんな人！
- 恋のキーワード
- あなたの恋愛スタイル
- こんな人と相性がいいよ
- あなたの魅力
- もっと素敵な恋をするためのヒント

### 表示ルール

- タイプ名は大きく表示
- コードは右上のバッジに小さく表示
- 相性が良いタイプは3件程度
- 魅力タグは3つ
- 恋のキーワードは3つ
- 恋愛スタイルは3ブロック
- 恋のヒントは1ブロック
- 最下部には固定文言を表示

- 恋のキーワード

- 相性が良いタイプ3件
